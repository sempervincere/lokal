### T-10: Field Hash Utility

- **Owner:** Dylan
- **Agent:** `typescript-pro`
- **Description:** Implement the canonical SHA-256 field hash per TDD Section 5.5 and CLAUDE.md Section 9. **This serialization is frozen** — changing it invalidates all existing on-chain hashes.
- **Deliverables:**
  - `src/lib/solana/fieldHash.ts`
- **Implementation (exact):**
  ```typescript
  export function computeFieldHash(fieldCode: string, value: unknown): string {
    const normalized = JSON.stringify(
      { fieldCode, value },
      Object.keys({ fieldCode, value }).sort(),
    );
    return crypto.createHash("sha256").update(normalized).digest("hex");
  }
  ```
- **Dependencies:** T-00
- **Acceptance Criteria:**
  - Same inputs always produce same output (deterministic)
  - `computeFieldHash("B1", { max_wtp: 28000 })` returns a 64-character hex string
  - Key order in input object does NOT affect output
  - Unit tests cover: same-input idempotency, different-value produces different hash, nested JSON value

### T-24: Helius Webhook — Payment Detection

- **Owner:** Dylan
- **Agent:** `backend-developer`
- **Skill:** `helius` (CRITICAL — webhook payload shape, HMAC verification, Enhanced Transactions format)
- **Description:** The webhook handler is the payment backbone. A bug here means no payments work. Implement per TDD Section 7 and CLAUDE.md Section 3, Flow B.
- **Deliverables:**
  - `src/app/api/webhooks/helius/route.ts`
- **Implementation steps (exact order):**
  1. Read raw body as text (needed for HMAC)
  2. Verify `helius-signature` header: `HMAC-SHA256(body, HELIUS_WEBHOOK_SECRET)` — return 401 if mismatch
  3. Parse body as JSON array of transactions
  4. For each tx: find `tokenTransfers` where `mint === IDRX_MINT_ADDRESS` AND `toUserAccount === PLATFORM_WALLET` AND `tokenAmount === 400000`
  5. Extract Memo: find instruction where `programId === "MemoSq4gqABAXKb96qnH8TysNcWxMyWCqXgDLGmfcHr"`, base64-decode `data` field
  6. Skip if sessionId is null/empty
  7. Deduplicate: skip if `session.solTxSignature` already set (UNIQUE constraint handles race)
  8. Update session: `status = PAYMENT_CONFIRMED`, `solTxSignature = tx.signature`
  9. Fire-and-forget: `generateReport(sessionId).catch(console.error)`
- **Dependencies:** T-01, T-04
- **Acceptance Criteria:**
  - Invalid HMAC returns 401 immediately
  - Valid payment activates session and triggers report generation
  - Duplicate `solTxSignature` is silently skipped (no error thrown)
  - sessionId correctly extracted from base64 Memo data
  - Always returns 200 to Helius (even on internal errors — prevents retry storms)
